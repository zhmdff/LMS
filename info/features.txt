University Academic Management System (UAMS)
1. Project Overview
UAMS is a centralized academic management platform designed for a single university.The system manages academic structure, student enrollment, curriculum selection, attendance tracking, grading, reporting, and formal academic requests.
This system is:
Academic-focused only
Rule-driven and structured
Designed for up to 10,000+ users
Built for real-world university deployment
Optimized for low-technical-knowledge users
Strictly role-based
Non-financial
Non-HR
Non-government integrated
No AI features


---
2. System Objectives
1. Digitize academic workflows.

2. Enforce strict attendance and grading rules.

3. Standardize curriculum registration.

4. Provide clear teacher dashboards for in-class usage.

5. Enable structured formal request workflows.

6. Generate academic reports.

7. Reduce manual paperwork.

7. Attendance Engine (Core Module)
This is a strict rule-driven system.
7.1 Time Logic
Opens 10 minutes before class start.
Closes 15 minutes after class start.
After 15 minutes:
i.e → can convert to q.b
q.b → cannot convert to i.e

If teacher does nothing → blank.

7.2 Attendance Status Types
i.e (iştirak edir)
q.b (qeyri mövcud)
q.iş (holiday)

7.3 Academic Rules
Attendance contributes 10 total points.
If student absent in 6 lessons → automatic fail.
Rule is system-enforced.
Configurable thresholds possible.

7.4 Correction Policy
Teacher cannot edit after window closes.
Teacher must submit correction letter inside system.
Admin approval required.
Full audit trail maintained.

7.5 Lecture vs Practice
Lecture may contain multiple groups.
Practice contains single group.
Unified attendance screen recommended for multi-group lectures.
Internally grouped but displayed combined.


---
8. Grading System
8.1 Structure
Each subject has:
Aktivlik balı (Activity)
Davamiyyət balı (Attendance)
İmtahan balı
Kollokvium balı
Sərbəst iş

Point structure is fixed across university.
8.2 Activity Scoring
Teacher assigns 0–10 per lesson.
System auto-sums.
Daily approval required.
Teacher cannot edit past scores.
Admin audit enabled.

8.3 Kollekvium
Created by teacher.
Active for defined period (e.g., 1 week).
Manually graded.
Added to subject total.

8.4 Exam
Often paper-based.
Results uploaded manually.
Online exams possible later.
Not MVP priority.


---
9. Failure Rules
A student fails a subject if:
Absent ≥ 6 lessons OR
Final score below threshold.

System auto-calculates status.
