# 1. Authentication Model

## No Registration

- Users are created by Admin.
- Login only (username + password).
- On login:
    - System detects role.
    - Redirects to role-specific layout.
    - User cannot switch roles.

---

# 2. Layout Architecture Overview

Each role has a separate layout structure:

1. Student Layout
2. Teacher Layout
3. Department Head Layout
4. University Admin Layout
5. Super Admin Layout
6. Tutor Layout (limited)

Each layout contains:

- Top bar
- Sidebar
- Main content
- Notifications (system alerts only)

---

### 4.3 Attendance (Core Page)

If current time within:

- 10 minutes before class
- 15 minutes after class start

Attendance automatically available.

Screen shows:

If Lecture:

- Combined list of all groups

If Practice:

- Single group list

Columns:

- Student name
- Status dropdown (i.e / q.b / q.iş)

Rules enforced automatically.

After time window:

- Screen locked.

If mistake:

- “Submit Correction Letter” button.

---

### 4.4 Activity Scores

Select:

- Subject
- Lesson date

Student list appears.

Enter:

- 0–10 points

Submit → Daily approval required.

Cannot edit after approval.

### 4.6 Grade Approval
